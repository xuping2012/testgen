## 一、背景与问题

### 1.1 背景

随着大语言模型（LLM）在软件测试领域的深入应用，基于AI的测试用例生成已成为质量保障的重要手段。然而，通用LLM在生成测试用例时存在以下痛点：

- **领域知识缺失**：通用模型缺乏企业特定业务逻辑、领域术语和规范要求
- **历史经验割裂**：过往测试用例、缺陷报告等经验资产未能有效复用
- **上下文孤立**：无法获取需求变更、代码实现等关联信息
- **结果不可控**：生成内容缺乏可解释性，难以追溯决策依据

### 1.2 核心问题

| 问题类型 | 具体表现 | 影响 |
|----------|----------|------|
| 需求理解偏差 | 生成的用例与实际需求不符 | 测试有效性降低 |
| 用例同质化 | 新增用例与历史用例高度重复 | 测试资源浪费 |
| 缺陷遗漏 | 边界条件、异常场景覆盖不足 | 质量隐患 |
| 可解释性差 | 无法说明用例生成依据 | 人工审核成本高 |

### 1.3 解决方案

引入RAG（Retrieval-Augmented Generation）架构，通过构建测试领域知识库，结合多路召回策略和置信度计算，实现：

- **知识增强**：融合历史用例、缺陷记录、业务规范等多源知识
- **精准召回**：多维度检索策略确保相关知识的高效获取
- **可信生成**：置信度计算和白盒化方案提升结果可解释性

### 4.2 需求匹配策略

#### 4.2.1 相似度计算公式

**语义相似度 (Semantic Similarity)**

```
语义相似度 Sim_semantic 基于向量的余弦相似度：

                    →    →
              A · B    Σ(aᵢ × bᵢ)
Sim_semantic = ───── = ──────────────
              |A||B|   √(Σaᵢ²) × √(Σbᵢ²)

其中：
  →    →                →
  A = 用户需求向量, B = 知识块向量
  aᵢ, bᵢ = 向量的第i维分量
```

**关键词覆盖度 (Keyword Coverage)**

```
关键词覆盖度 Cov_keyword 计算：

           |keywords_user ∩ keywords_chunk|
Cov_keyword = ───────────────────────────── × weight_exact
              |keywords_user|

权重加成：
  weight_exact = 1.0 (完全匹配)
  weight_fuzzy = 0.6 (模糊匹配/同义词)
  weight_partial = 0.3 (部分包含)

最终覆盖度 = Σ(匹配类型权重 × 匹配词数) / 总词数
```

**模块归属度 (Module Affinity)**

```
模块归属度 Aff_module 考虑层级匹配：

Aff_module = match_level / total_level

层级匹配规则：
  精确匹配: match_level += 4
  二级匹配: match_level += 3
  一级匹配: match_level += 2
  父级匹配: match_level += 1
  无匹配:   match_level += 0
```

#### 4.2.2 综合相似度公式

```
综合相似度 Sim_total = w₁×Sim_semantic + w₂×Cov_keyword + w₃×Aff_module + w₄×Recency

权重配置：
  w₁ = 0.40  (语义相似度)
  w₂ = 0.25  (关键词覆盖)
  w₃ = 0.20  (模块归属)
  w₄ = 0.15  (时效性加成)

时效性加成 Recency：
  最近3个月: Recency = 1.0
  3-6个月:   Recency = 0.8
  6-12个月:  Recency = 0.6
  12个月以上: Recency = 0.4
```

```python
def match_test_point_level(query: str, candidate_case: dict) -> MatchResult:
    """
    测试点级别匹配算法
    
    匹配维度:
    1. 功能点匹配
    2. 操作步骤匹配
    3. 预期结果匹配
    4. 边界条件匹配
    """
    
    # 1. 功能点匹配
    func_score = calculate_function_similarity(
        query.extract_function_points(),
        candidate_case.test_steps
    )
    
    # 2. 操作步骤模式匹配
    step_score = calculate_step_pattern_similarity(
        query.extract_operation_pattern(),
        candidate_case.test_steps
    )
    
    # 3. 预期结果类型匹配
    result_score = calculate_result_type_similarity(
        query.expected_result_type,
        candidate_case.expected_result
    )
    
    # 4. 边界条件匹配
    boundary_score = calculate_boundary_similarity(
        query.boundary_conditions,
        candidate_case.edge_cases
    )
    
    # 综合测试点得分
    test_point_score = (
        0.35 * func_score +      # 功能点权重
        0.30 * step_score +      # 操作步骤权重
        0.20 * result_score +    # 预期结果权重
        0.15 * boundary_score   # 边界条件权重
    )
    
    return MatchResult(
        total_score=test_point_score,
        breakdown={
            'function': func_score,
            'step': step_score,
            'result': result_score,
            'boundary': boundary_score
        },
        match_level='high' if test_point_score > 0.7 else 'medium' if test_point_score > 0.5 else 'low'
    )
```

```
┌────────────────┬────────────────────────────────┐
│ 匹配级别       │ 得分                          │
├────────────────┼────────────────────────────────┤

│ 无匹配         │ 0.0                           │
└────────────────┴────────────────────────────────┘

计算公式：
Score_module = max(各层级匹配得分) × domain_weight

其中 domain_weight = 1.2 (高风险模块) / 1.0 (普通模块) / 0.8 (低风险模块)
```

**路径2: 关键词匹配验证**

```
关键词匹配得分 (Keyword Match Score):

精确匹配得分:
  Score_exact = (精确匹配关键词数 / 总查询关键词数) × 2.0

模糊匹配得分:
  Score_fuzzy = (模糊匹配关键词数 / 总查询关键词数) × 1.0

语义相似词匹配:
  Score_semantic = (语义匹配词数 / 总查询关键词数) × 0.6

综合关键词得分:
Score_keyword = min(Score_exact + Score_fuzzy + Score_semantic, 1.0)
```

**路径3: 代码关联验证**

```
代码关联得分 (Code Correlation Score):

关联维度：
1. 接口/函数签名匹配
2. 参数类型匹配
3. 返回值类型匹配
4. 业务逻辑相似度

计算公式：
Score_code = w₁×API_match + w₂×param_match + w₃×return_match + w₄×logic_sim

权重配置：
  w₁ = 0.30 (接口匹配)
  w₂ = 0.20 (参数匹配)
  w₃ = 0.15 (返回值匹配)
  w₄ = 0.35 (业务逻辑相似度)

匹配算法：
  API_match = 1 if 函数名完全匹配 else 0.5 if 包含匹配 else 0
  param_match = 匹配参数数 / 总参数数
  logic_sim = 代码结构的语义嵌入余弦相似度
```

```
  weakly_related    → 0.4
  unrelated         → 0.0
```

#### 4.5.2 置信度分级输出

```python
class ConfidenceLevel(Enum):
    """置信度等级枚举"""
    A_HIGH = "A"      # 高置信度
    B_MEDIUM_HIGH = "B"  # 中高置信度
    C_MEDIUM = "C"    # 中置信度
    D_LOW = "D"       # 低置信度
    E_INVALID = "E"   # 无效

def calculate_confidence(score: float, scenario: str) -> ConfidenceResult:
    """
    根据场景计算置信度等级
    
    Args:
        score: 原始得分
        scenario: 使用场景
    
    Returns:
        ConfidenceResult: 置信度结果
    """
    
    # 场景对应的阈值配置
    THRESHOLD_CONFIG = {
        'function_test': {'A': 0.85, 'B': 0.70, 'C': 0.50, 'D': 0.30},
        'regression': {'A': 0.80, 'B': 0.65, 'C': 0.45, 'D': 0.25},
        'boundary': {'A': 0.75, 'B': 0.60, 'C': 0.40, 'D': 0.20},
        'smoke': {'A': 0.90, 'B': 0.75, 'C': 0.55, 'D': 0.35},
    }
    
    thresholds = THRESHOLD_CONFIG.get(scenario, THRESHOLD_CONFIG['function_test'])
    
    if score >= thresholds['A']:
        return ConfidenceResult(level=ConfidenceLevel.A_HIGH, action='直接采纳')
    elif score >= thresholds['B']:
        return ConfidenceResult(level=ConfidenceLevel.B_MEDIUM_HIGH, action='轻度审核后采纳')
    elif score >= thresholds['C']:
        return ConfidenceResult(level=ConfidenceLevel.C_MEDIUM, action='标注来源，人工确认')
    elif score >= thresholds['D']:
        return ConfidenceResult(level=ConfidenceLevel.D_LOW, action='仅参考，全量审核')
    else:
        return ConfidenceResult(level=ConfidenceLevel.E_INVALID, action='过滤不采纳')
```

```python
def build_context_knowledge(knowledge_chunks: List[Dict], priority: str = "high") -> str:
    """
    构建知识注入上下文
    
    Args:
        knowledge_chunks: 知识块列表
        priority: 注入优先级 high/medium/low
    
    Returns:
        格式化后的知识上下文字符串
    """
    
    if priority == "high":
        # 强制注入的内容
        context_parts = [
            "## 强制参考信息",
            format_mandatory_knowledge(knowledge_chunks)
        ]
    elif priority == "medium":
        # 包含建议性内容
        context_parts = [
            "## 建议参考信息",
            format_suggested_knowledge(knowledge_chunks)
        ]
    else:
        # 包含可选内容
        context_parts = [
            "## 可选参考信息",
            format_optional_knowledge(knowledge_chunks)
        ]
    
    return "\n\n".join(context_parts)


def format_mandatory_knowledge(chunks: List[Dict]) -> str:
    """格式化强制知识"""
    lines = []
    
    # 用例规范
    if case_standards := [c for c in chunks if c['type'] == 'standard']:
        lines.append("### 测试用例规范")
        for s in case_standards[:2]:
            lines.append(f"- {s['content']}")
    
    # 术语定义
    if glossary := [c for c in chunks if c['type'] == 'glossary']:
        lines.append("\n### 术语定义")
        for g in glossary[:5]:
            lines.append(f"- **{g['term']}**: {g['definition']}")
    
    # 安全要求
    if safety := [c for c in chunks if c['type'] == 'safety']:
        lines.append("\n### 安全要求")
        for s in safety[:3]:
            lines.append(f"- {s['content']}")
    
    return "\n".join(lines)
```

### 6.4 Prompt质量优化

#### 6.4.1 Token控制策略

```python
class PromptTokenController:
    """Prompt Token控制器"""
    
    def __init__(self, max_tokens: int = 4096):
        self.max_tokens = max_tokens
        # Token预算分配
        self.budget = {
            'system': 500,      # 系统指令
            'role': 300,        # 角色设定
            'context': 2000,    # 知识上下文
            'requirement': 500, # 用户需求
            'output_format': 500,  # 输出格式
            'safety': 200,      # 安全约束
        }
    
    def optimize_prompt(self, prompt_components: Dict) -> str:
        """优化Prompt，控制在Token预算内"""
        
        optimized_components = {}
        total_tokens = 0
        
        for component_name, content in prompt_components.items():
            target_tokens = self.budget.get(component_name, 500)
            truncated_content = self.truncate_to_tokens(content, target_tokens)
            optimized_components[component_name] = truncated_content
            total_tokens += estimate_tokens(truncated_content)
        
        # 如果超限，进行二次压缩
        if total_tokens > self.max_tokens:
            optimized_components = self.compress_prompt(optimized_components)
        
        return self.assemble_prompt(optimized_components)
    
    def truncate_to_tokens(self, content: str, max_tokens: int) -> str:
        """截断内容到指定Token数"""
        tokens = tokenize(content)
        if len(tokens) <= max_tokens:
            return content
        
        # 按优先级保留关键部分
        truncated_tokens = tokens[:max_tokens]
        return detokenize(truncated_tokens)
    
    def compress_prompt(self, components: Dict) -> Dict:
        """压缩Prompt以符合Token限制"""
        
        # 压缩策略：按优先级降低内容长度
        compress_ratios = {
            'context': 0.7,      # 上下文压缩70%
            'output_format': 1.0,  # 格式保持不变
            'requirement': 1.0,    # 需求保持不变
        }
        
        compressed = {}
        for name, content in components.items():
            ratio = compress_ratios.get(name, 0.8)
            target_tokens = int(self.budget.get(name, 500) * ratio)
            compressed[name] = self.truncate_to_tokens(content, target_tokens)
        
        return compressed
```

```python
def validate_output_format(output: str, output_type: str) -> ValidationResult:
    """校验输出格式"""
    
    rules = OUTPUT_FORMAT_RULES.get(output_type, {})
    
    # 字段完整性检查
    for field in rules.get("required_fields", []):
        if field not in output:
            return ValidationResult(
                valid=False,
                errors=[f"Missing required field: {field}"]
            )
    
    # 格式模式检查
    for field, pattern in rules.get("field_patterns", {}).items():
        if not re.match(pattern, extract_field(output, field)):
            return ValidationResult(
                valid=False,
                errors=[f"Field {field} format mismatch: {pattern}"]
            )
    
    # 结构规则检查
    for field, rule in rules.get("structure_rules", {}).items():
        count = count_field_occurrences(output, field)
        
        if "min_count" in rule and count < rule["min_count"]:
            return ValidationResult(
                valid=False,
                errors=[f"Field {field} count {count} < min {rule['min_count']}"]
            )
        
        if "max_count" in rule and count > rule["max_count"]:
            return ValidationResult(
                valid=False,
                errors=[f"Field {field} count {count} > max {rule['max_count']}"]
            )
    
    return ValidationResult(valid=True, errors=[])
```

#### 8.3.1 召回效果评估

```python
# 召回效果评估指标

EVALUATION_METRICS = {
    # 检索相关指标
    "retrieval_metrics": {
        "recall@k": "Top-K召回率",
        "precision@k": "Top-K精确率",
        "ndcg@k": "归一化折扣收益",
        "mrr": "平均倒数排名"
    },
    
    # 生成质量指标
    "generation_metrics": {
        "citation_accuracy": "引用准确率",
        "coverage_rate": "需求覆盖率",
        "redundancy_rate": "冗余率",
        "human_preference_score": "人工偏好评分"
    },
    
    # 置信度校准
    "calibration_metrics": {
        "calibration_error": "校准误差 (ECE)",
        "confidence_coverage": "置信度-覆盖一致性"
    }
}

def evaluate_retrieval_quality(
    test_set: List[TestQuery], 
    retrieval_results: Dict
) -> EvaluationReport:
    """
    评估检索质量
    
    评估维度:
    1. 检索召回率 (是否能找到相关知识)
    2. 排序质量 (最相关的是否排在前)
    3. 置信度校准 (置信度是否与实际准确率一致)
    """
    
    results = {}
    
    # 1. 计算召回率
    recall_scores = []
    for query in test_set:
        expected_docs = set(query.relevant_docs)
        retrieved_docs = set(retrieval_results[query.id][:10])  # Top-10
        
        recall = len(expected_docs & retrieved_docs) / len(expected_docs)
        recall_scores.append(recall)
    
    results['recall@10'] = mean(recall_scores)
    
    # 2. 计算NDCG
    ndcg_scores = []
    for query in test_set:
        ndcg = calculate_ndcg(
            retrieval_results[query.id],
            query.relevant_docs,
            k=10
        )
        ndcg_scores.append(ndcg)
    
    results['ndcg@10'] = mean(ndcg_scores)
    
    # 3. 置信度校准
    calibration_error = calculate_ece(
        predicted_confidences=[r['confidence'] for r in retrieval_results],
        actual_accuracies=[r['is_correct'] for r in retrieval_results],
        n_bins=10
    )
    
    results['calibration_error'] = calibration_error
    
    return EvaluationReport(results)
```
